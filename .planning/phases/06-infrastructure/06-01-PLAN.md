  <name>Task 1: Dump SQLite data and start PostgreSQL</name>
  <files>backend/data_dump.json</files>
  <action>
Step 1: Dump all data from the SQLite database using Django's dumpdata command. Run from the backend directory with base settings (which default to SQLite):

```bash
cd backend
DJANGO_SETTINGS_MODULE=config.settings.base uv run python manage.py dumpdata \
  --exclude contenttypes \
  --exclude auth.permission \
  --exclude admin.logentry \
  --exclude sessions.session \
  --natural-primary \
  --natural-foreign \
  --indent 2 \
  -o data_dump.json
```

The exclusions are critical -- contenttypes and permissions are auto-created by `migrate` and loading them would cause IntegrityError duplicates.

Step 2: Verify the dump file exists and has content. Check it contains objects (should be several hundred given 770KB DB):

```bash
uv run python -c "import json; d=json.load(open('data_dump.json')); print(f'{len(d)} objects dumped')"
```

Step 3: Start PostgreSQL and Redis via Docker Compose:

```bash
cd ..  # back to project root
docker-compose up -d db redis
```

Wait for the healthcheck to pass (pg_isready). Verify PostgreSQL is accepting connections:

```bash
docker-compose exec db pg_isready -U couplesai
```

Step 4: Run Django migrations against PostgreSQL to create the schema. Use production settings with the Docker Compose DATABASE_URL. The FERNET_KEYS must match the value from backend/.env (default: `Tw6IKPg_AApN_RVtLWwmCGM20q3sUkURg7LwWvjClSA=`):

```bash
cd backend
DATABASE_URL=postgres://couplesai:couplesai_password@localhost:5432/couplesai \
DJANGO_SETTINGS_MODULE=config.settings.production \
DJANGO_SECRET_KEY=migration-temp-key \
FERNET_KEYS=Tw6IKPg_AApN_RVtLWwmCGM20q3sUkURg7LwWvjClSA= \
ALLOWED_HOSTS=localhost \
uv run python manage.py migrate --noinput
```

All migrations must pass without errors. If any migration uses SQLite-specific SQL, it will fail here -- diagnose and fix.
