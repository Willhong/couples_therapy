  <name>Task 1: Configure app.json for push notifications and EAS</name>
  <files>mobile/app.json</files>
  <action>
Update `mobile/app.json` to add all configuration required for push notifications on Android via Expo.

The current app.json is minimal (name, slug, version, newArchEnabled, web, experiments). Add:

1. **plugins**: Add `"expo-notifications"` to enable the notification native module
2. **android.package**: Set to `"com.couplesai.app"` (matches existing build.gradle)
3. **android.googleServicesFile**: Set to `"./google-services.json"` (will be placed by user)
4. **android.useNextNotificationsApi**: Set to `true` (required for SDK 54+ FCM V1)
5. **extra.eas.projectId**: Set to a placeholder `"YOUR_EAS_PROJECT_ID"` that will be filled after `eas init`
6. **scheme**: Set to `"couplesai"` (already used for deep links, ensures notification deep links work)
7. **icon**: Keep existing if set, or add `"./assets/icon.png"` as placeholder
8. **splash**: Add basic splash config if not present

The full resulting app.json should look like:

```json
{
  "expo": {
    "name": "CouplesAI",
    "slug": "couples-ai",
    "version": "1.0.0",
    "scheme": "couplesai",
    "newArchEnabled": true,
    "plugins": [
      "expo-notifications"
    ],
    "android": {
      "package": "com.couplesai.app",
      "googleServicesFile": "./google-services.json",
      "useNextNotificationsApi": true
    },
    "extra": {
      "eas": {
        "projectId": "YOUR_EAS_PROJECT_ID"
      }
    },
    "web": {
      "favicon": "./assets/favicon.png",
      "bundler": "metro"
    },
    "experiments": {
      "typedRoutes": true
    }
  }
}
```

Do NOT add iOS-specific configuration yet -- iOS push is deferred to Phase 9 per research recommendation (no Apple Developer Account).

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Firebase project creation and EAS project linking</name>
  <action>
The following steps require manual action because they involve external service authentication that Claude cannot perform.

**Part A: Firebase Project Setup**

1. Go to https://console.firebase.google.com/
2. Click "Add project" (or open existing project if one exists for CouplesAI)
3. Name the project "CouplesAI" (or similar)
4. Enable Google Analytics if desired (optional)
5. Once project is created, go to **Project settings** (gear icon)
6. Under **General** tab, scroll to "Your apps"
7. Click **Add app** -> **Android**
8. Package name: `com.couplesai.app`
9. App nickname: `CouplesAI`
10. Click **Register app**
11. **Download `google-services.json`**
12. Place the file at `mobile/google-services.json`

**Part B: EAS Project Linking**

1. In terminal, navigate to the mobile directory:
   ```bash
   cd mobile
   ```
2. Login to Expo (if not already):
   ```bash
   npx eas login
   ```
3. Initialize the EAS project:
   ```bash
   npx eas init
   ```
   This will output a **projectId** (UUID format). Copy it.

4. Open `mobile/app.json` and replace `"YOUR_EAS_PROJECT_ID"` with the actual projectId.

**Part C: FCM V1 Credentials (for EAS Build)**

1. Still in the mobile directory:
   ```bash
   npx eas credentials --platform android
   ```
2. Select "Push Notifications: Manage your FCM V1 API Key"
3. Follow prompts to upload the Firebase service account key
   - In Firebase console -> Project settings -> Service accounts -> Generate new private key
   - Upload the downloaded JSON file when EAS prompts for it

**After completing all steps, type "done" to continue.**
