<task type="auto">
  <name>Task 3: Create database schema with RLS policies</name>
  <files>
    - supabase/migrations/001_initial_schema.sql
  </files>
  <action>
Create `supabase/migrations/001_initial_schema.sql` with:

1. **profiles table** (extends auth.users):
   - id UUID (FK to auth.users)
   - display_name VARCHAR
   - onboarding_completed_at TIMESTAMPTZ
   - tutorial_completed_at TIMESTAMPTZ
   - disclaimer_accepted_at TIMESTAMPTZ
   - disclaimer_version VARCHAR
   - created_at, updated_at timestamps
   - RLS: Users can only read/update own profile

2. **couples table**:
   - id UUID PK
   - user1_id, user2_id UUID (FKs to auth.users)
   - status VARCHAR ('pending', 'active', 'disconnected')
   - connected_at TIMESTAMPTZ
   - created_at timestamp
   - RLS: Users can see couples they're part of
   - Reference 01-RESEARCH.md Pattern 4

3. **invite_codes table**:
   - id UUID PK
   - code VARCHAR(6) UNIQUE
   - creator_id UUID (FK to auth.users)
   - expires_at TIMESTAMPTZ (default NOW() + 24 hours)
   - used_by UUID, used_at TIMESTAMPTZ
   - created_at timestamp
   - Index on code for fast lookup
   - RLS: Users can view own codes, create new codes
   - Reference 01-RESEARCH.md Pattern 3

4. **recording_consents table**:
   - id UUID PK
   - couple_id UUID (FK to couples)
   - initiated_by UUID (FK to auth.users)
   - user1_consented_at, user2_consented_at TIMESTAMPTZ
   - status VARCHAR ('pending', 'approved', 'declined')
   - created_at timestamp
   - RLS: Only couple members can see/create

5. **Helper function** `get_couple_id()`:
   - Returns active couple_id for current user
   - SECURITY DEFINER for use in RLS policies

6. **Helper function** `generate_invite_code()`:
   - Generates 6-char alphanumeric code
   - Excludes confusing chars (0/O, 1/I/L)

CRITICAL: Always `ALTER TABLE x ENABLE ROW LEVEL SECURITY` before creating policies.
  </action>
  <verify>
    - SQL syntax is valid (no parsing errors)
    - All tables have RLS enabled
    - All tables have appropriate policies
  </verify>

    - Helper functions ready for invite code generation
  </done>
