Create `src/utils/inviteCode.ts`:

**Functions:**
- `createInviteCode(supabase)`:
  - Call `supabase.rpc('generate_invite_code')` to get code
  - Insert into invite_codes table with creator_id, expires_at (24 hours)
  - Return the 6-digit code string

- `validateInviteCode(supabase, code)`:
  - Query invite_codes table by code (case-insensitive, toUpperCase)
  - Check: exists, not used (used_by is null), not expired
  - Return { valid: boolean, creatorId?: string, error?: string }

- `redeemInviteCode(supabase, code)`:
  - Validate code first
  - Create couples record (user1_id = creator, user2_id = current user, status = 'active')
  - Update invite_codes: set used_by, used_at
  - Return { success: boolean, coupleId?: string, error?: string }

Reference 01-RESEARCH.md Pattern 3 for implementation.

Create `src/utils/deepLink.ts`:

**Functions:**
- `createInviteLink(code)`:
  - Use Linking.createURL('invite', { queryParams: { code } })
  - Returns: couplesai://invite?code=ABC123

- `parseInviteLink(url)`:
  - Use Linking.parse(url)
  - Extract code from queryParams if hostname is 'invite'
  - Return code string or null

**Hook:**
- `useInviteLink()`:
  - Use Linking.useURL() to detect incoming links
  - Parse link and extract code
  - Navigate to partner-link screen with code param

Reference 01-RESEARCH.md Pattern 5 for implementation.

Update `app.json`:
- Ensure "scheme": "couplesai" is set
- Add Android intentFilters for couplesai://invite
- Add iOS URL types configuration

Create `src/hooks/usePartner.ts`:

**State:**
- `couple`: Current couple record (or null)
- `partner`: Partner's profile data (or null)
- `connectionStatus`: 'none' | 'pending' | 'active' | 'disconnected'
- `loading`: Boolean for initial load
- `myInviteCode`: Current user's active invite code (if any)

**Methods:**
- `generateInviteCode()`: Create new code, store in myInviteCode state
- `enterCode(code)`: Validate and redeem partner's code
- `getInviteLink()`: Get shareable link for current code
- `disconnect()`: Set couple status to 'disconnected' (keep record for data integrity)
- `refresh()`: Reload partner/couple data

**Implementation:**

- Subscribe to couples table changes with Supabase realtime (optional, nice-to-have)
- Handle edge cases:
