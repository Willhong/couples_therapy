Create `mobile/src/hooks/usePartner.ts`:

**Types:**
```typescript
interface Partner {
  id: number;
  email: string;
}

interface Couple {
  id: number;
  status: 'pending' | 'active' | 'disconnected';
  connected_at: string | null;
  partner: Partner | null;
}

interface InviteCode {
  code: string;
  expires_at: string;
  deep_link: string;
}

type ConnectionStatus = 'none' | 'pending' | 'active' | 'disconnected';
```

**State:**
- couple: Couple | null
- connectionStatus: ConnectionStatus
- myInviteCode: InviteCode | null
- loading: boolean
- error: string | null

**Methods:**

`generateInviteCode(): Promise<InviteCode>`
- POST to /couples/invite/generate/
- Store response in myInviteCode state
- Return the invite code object

`enterCode(code: string): Promise<void>`
- POST to /couples/invite/redeem/ with { code }
- On success: update couple state, set connectionStatus to 'active'
- On error: set error state with Korean message:
  - "유효하지 않은 코드입니다" (invalid)
  - "코드가 만료되었습니다" (expired)
  - "이미 사용된 코드입니다" (already used)
  - "이미 파트너와 연결되어 있습니다" (already connected)
  - "본인의 코드는 사용할 수 없습니다" (own code)

`getInviteLink(): string | null`
- Return myInviteCode?.deep_link or null

`disconnect(): Promise<void>`
- POST to /couples/disconnect/
- Update couple status to 'disconnected'
- Clear partner info

`refresh(): Promise<void>`
- GET /couples/
- Update couple and connectionStatus based on response
- If active couple exists: connectionStatus = 'active', fetch partner info
- If no couple: connectionStatus = 'none'

**Provider:**
- Create PartnerContext and PartnerProvider
- On mount: call refresh() to load current partner status
- Provide state and methods via context

`usePartner(): PartnerContextType`
- Return context value
- Throw error if used outside provider

**Integration:**
- Add PartnerProvider to app _layout.tsx (inside AuthProvider)
