<decisions>
## Implementation Decisions

### 온보딩 흐름
- 순서: 가입 → 면책조항 → 파트너 연결 (스크리닝 제거)
- 솔로 모드: 제한적 허용 (일부 기능만 파트너 없이 사용 가능)
- 중단 처리: 끝난 단계부터 자동 재개
- 진행 표시: 프로그레스 바
- 뒤로가기: 허용 (이전 단계로 돌아가서 수정 가능)
- 오류 처리: 인라인 안내 (같은 화면에서 오류 메시지 + 재시도 버튼)

### 온보딩 완료 후
- 첫 화면: 튜토리얼 (코치마크 투어 형식)
- 튜토리얼: 필수 시청 (건너뛰기 불가, 설정에서 다시 볼 수 있음)
- 파트너 튜토리얼: 파트너도 동일한 튜토리얼 필수 시청

### 톤 및 비주얼
- 톤: 전문적이고 신뢰감 있는 (깨끗한 디자인, 정제된 언어)
- 비주얼: 각 단계마다 관련 일러스트 표시

### 파트너 연결 방식
- 초대 방식: 코드 + 링크 둘 다 지원
- 코드: 6자리 코드 생성 후 공유
- 링크: 고유 URL 생성해서 메신저/카톡으로 전송
- 유효기간: 24시간 (만료 시 재생성)
- 연결 알림: 푸시 알림 + 앱 내 알림 둘 다
- 거절/무응답: 새 코드 생성해서 재초대 가능

### 동의 및 면책 표시
- 면책조항: 가입 폼에 체크박스로 포함 ("전문 상담 대체 불가" 동의)
- 녹음 동의: 매번 녹음 시 요청, 한쪽에서 요청하고 양측 동의 시에만 진행
- 동의 거절 시: 녹음 불가 (텍스트 입력으로만 진행)
- 동의 이력: 설정 메뉴에서 동의 내역 조회 가능

### Claude's Discretion
- 프로그레스 바 디자인
- 일러스트 스타일
- 코치마크 투어 상세 UX
- 오류 메시지 문구

</decisions>

<specifics>
## Specific Ideas

- 전문적이고 신뢰감 있는 톤 — 부드러운 것보다 깨끗하고 정제된 느낌
- 녹음은 양측 동의 필수 — 한쪽에서 요청, 상대방이 승인해야 시작
- 파트너 없이도 제한적 사용 가능 — 자기 성찰 목적

</specifics>

<deferred>
## Deferred Ideas

- **학대 스크리닝** — 온보딩에서 제거, Phase 2 이후 "사용 중 패턴 감지 + 리소스 안내" 방식으로 전환
  - 이유: 온보딩 시점에 신뢰 부족, 가해자 거짓 응답 가능, 피해자 인식 부재, UX 저해

</deferred>
