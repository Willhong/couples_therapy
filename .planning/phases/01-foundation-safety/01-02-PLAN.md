Create `mobile/src/lib/api.ts`:
- Reference 01-RESEARCH.md Common Pitfalls section for interceptor pattern
- Export API_URL from process.env.EXPO_PUBLIC_API_URL (default: 'http://localhost:8000')
- Create axios instance with baseURL: `${API_URL}/api/v1`
- Request interceptor:
  - Get access token from TokenStorage
  - Add Authorization: Bearer {token} header if token exists
- Response interceptor:
  - On 401 error:
    - Get refresh token from TokenStorage
    - POST to /auth/token/refresh/ with { refresh: refreshToken }
    - If successful: save new tokens, retry original request
    - If failed: clear tokens, reject with error
  - Pass through other errors

Export:
- api: configured axios instance
- API_URL: base URL constant

Create `mobile/src/hooks/useAuth.ts`:

**AuthContext and Provider:**
- Create AuthContext with: user, loading, signUp, signIn, signOut, refreshUser
- AuthProvider component that:
  - On mount: check for existing tokens, verify with /auth/user/ endpoint
  - Provide auth state and methods to children

**Types:**
```typescript
interface User {
  id: number;
  email: string;
  disclaimer_accepted: boolean;
  onboarding_completed: boolean;
  tutorial_completed: boolean;
}

interface AuthContextType {
  user: User | null;
  loading: boolean;
  signUp: (email: string, password: string, disclaimerVersion: string) => Promise<void>;
  signIn: (email: string, password: string) => Promise<void>;
  signOut: () => Promise<void>;
  refreshUser: () => Promise<void>;
}
```

**Methods:**
- signUp: POST to /auth/registration/ with email, password1, password2, disclaimer_accepted: true, disclaimer_version
  - On success: save tokens, set user state
- signIn: POST to /auth/token/ with email, password
  - On success: save tokens, fetch user from /auth/user/, set user state
- signOut: POST to /auth/logout/ (optional), clear tokens, clear user state
- refreshUser: GET /auth/user/, update user state
