  <action>
Create `src/hooks/useAuth.ts` that provides:

**State:**
- `session`: Current Supabase session (or null)
- `user`: Current user object (or null)
- `loading`: Boolean for initial session check
- `profile`: User profile data from profiles table

**Methods:**
- `signUp(email, password, disclaimerVersion)`: Create account with disclaimer metadata
  - Pass disclaimer_accepted_at (ISO timestamp) and disclaimer_version in options.data
  - After signup, insert row into profiles table with disclaimer info
- `signIn(email, password)`: Email/password login
- `signOut()`: Clear session
- `refreshSession()`: Force session refresh

**Implementation details:**
- Use `supabase.auth.onAuthStateChange` to track session changes
- Load profile data after session established
- Handle loading state during initial session check
- Clean up listener on unmount

Reference 01-RESEARCH.md for Supabase auth patterns.
