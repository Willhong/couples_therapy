Create `mobile/src/hooks/useRecordingConsent.ts`:

**WebSocket connection:**
- Connect to ws://{API_HOST}/ws/consent/?token={accessToken}
- Handle connection open/close/error
- Parse incoming JSON messages

**State:**
- consentRequest: { session_id, status, requester_id, expires_at } | null
- myConsent: boolean
- partnerConsent: boolean | null
- partnerOnline: boolean
- status: 'idle' | 'pending' | 'approved' | 'declined'
- error: string | null

**Methods:**
- connect(): void
  - Open WebSocket with JWT token in query string
  - Set up message handlers

- disconnect(): void
  - Close WebSocket connection

- initiateConsent(): void
  - Generate session_id (uuid)
  - Send { action: 'request_consent', session_id }
  - Set myConsent = true, status = 'pending'

- giveConsent(): void
  - Send { action: 'respond_consent', session_id, consented: true }
  - Set myConsent = true
  - Check if both consented -> status = 'approved'

- denyConsent(): void
  - Send { action: 'respond_consent', session_id, consented: false }
  - Set status = 'declined'

- reset(): void
  - Clear state, ready for new consent request

**Message handlers:**
- consent_requested: Set consentRequest, show prompt to responder
- consent_updated: Update partnerConsent, status
- user_joined: Set partnerOnline = true
- user_left: Set partnerOnline = false

**Reconnection:**
- On disconnect: attempt reconnect with exponential backoff
- Max 5 attempts, then show error
