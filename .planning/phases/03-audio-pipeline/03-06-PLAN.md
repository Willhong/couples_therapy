    2. Create backend/apps/patterns/models.py:
       - Pattern model:
         - id: UUIDField primary key
         - user: FK to AUTH_USER_MODEL
         - couple: FK to couples.Couple (nullable)
         - conversation: FK to chat.Conversation (the session where pattern was found)
         - pattern_type: CharField choices ('trigger_phrase', 'recurring_topic', 'escalation')
         - content: TextField (the detected pattern text, e.g. "넌 항상 늦어")
         - category: CharField (e.g. "finance", "time", "household", "communication", nullable)
         - severity: IntegerField (1-5, how strong/frequent)
         - context_snippet: TextField (surrounding text for context)
         - created_at: DateTimeField
         - db_table = 'patterns'

       - InsightSummary model (per-session insights):
         - id: UUIDField primary key
         - conversation: OneToOneField to chat.Conversation
         - user: FK to AUTH_USER_MODEL
         - trigger_phrases: JSONField (list of detected trigger phrases with positions)
         - recurring_topics: JSONField (list of topic matches with prior sessions)
         - escalation_score: IntegerField (0-10, how escalated the conversation got)
         - ai_summary: EncryptedTextField (AI-generated insight text)
         - created_at: DateTimeField
         - db_table = 'insight_summaries'

       - WeeklySummary model:
         - id: UUIDField primary key
         - user: FK to AUTH_USER_MODEL
         - couple: FK to couples.Couple (nullable)
         - period_start: DateField
         - period_end: DateField
         - session_count: IntegerField
         - top_topics: JSONField (list of {topic, count})
         - trigger_frequency: JSONField ({phrase: count})
         - trend_text: EncryptedTextField (AI-generated trend description)
         - escalation_trend: CharField ('improving', 'stable', 'worsening')
         - created_at: DateTimeField
         - db_table = 'weekly_summaries'

    3. Create backend/apps/patterns/admin.py with basic registrations

    4. Create backend/apps/patterns/services/detector.py:
       - `detect_patterns(conversation_id: str) -> dict`:
         a. Load conversation messages (text chat) or transcript segments (audio)
         b. Gather all text content into a single block
         c. Call LLM (using get_chat_model from chat.services.llm_service) with PATTERN_DETECTION_PROMPT:
            System prompt asks LLM to analyze Korean text for:
            - Trigger phrases: absolutist language ("넌 항상", "넌 절대", "맨날", "왜 항상")
            - Recurring topics: categorize the main conflict topic
            - Escalation patterns: detect if intensity increases through the conversation
            Return as JSON: { trigger_phrases: [{text, context}], topics: [{name, category}], escalation_score: int }
         d. Parse LLM response (reuse _parse_json_response pattern from reframing_graph.py)
         e. Save Pattern records for each trigger phrase and topic
         f. Create/update InsightSummary for this conversation

       - `find_recurring_topics(user_id: str, new_topics: list) -> list`:
         a. Query existing Pattern records for this user
         b. Match new_topics against historical topics by category
         c. Return matches with count (e.g. "돈 문제로 4번째 대화")

       - PATTERN_DETECTION_PROMPT (Korean):
         ```
         당신은 부부 갈등 패턴 분석가입니다. 아래 대화 내용을 분석하세요.

         ## 분석 항목
         1. **트리거 문구**: 절대적 표현 ("넌 항상", "넌 절대", "맨날 그래"), 비난 표현 ("왜 그런 식으로"), 일반화 표현
         2. **갈등 주제**: 주요 갈등 주제를 카테고리로 분류 (돈/재정, 가사, 시간, 의사소통, 가족, 육아, 기타)
         3. **에스컬레이션**: 대화가 진행되면서 감정 강도가 어떻게 변하는지 (0-10 점수)

         ## 응답 형식 (JSON)
         {
           "trigger_phrases": [{"text": "감지된 문구", "context": "주변 맥락 1-2문장"}],
           "topics": [{"name": "주제 이름", "category": "카테고리"}],
           "escalation_score": 0-10,
           "summary": "이 대화의 핵심 패턴 요약 (2-3문장, 중립적 관찰 톤)"
         }

         ## 톤
         - 중립적이고 관찰적인 톤 유지
         - 판단이나 비난 없이 사실 기반 분석
         - 한국어로 응답
         ```

    5. Create backend/apps/patterns/services/summarizer.py:
       - `generate_weekly_summary(user_id: str, period_start, period_end) -> dict`:
         a. Gather all Pattern records for this user in the period
         b. Aggregate: top topics by frequency, trigger phrase frequency, escalation trend
         c. Call LLM with WEEKLY_SUMMARY_PROMPT to generate natural language summary
         d. Create WeeklySummary record
         e. Return summary data

       - WEEKLY_SUMMARY_PROMPT: asks LLM to write a brief, neutral weekly summary in Korean
         Example output: "이번 주에 3번의 대화가 있었어요. 재정 관련 주제가 가장 많았고, '넌 항상'이라는 표현이 2번 사용되었어요. 전반적으로 감정 강도는 지난주보다 안정적이에요."

         For all active couples (or specific couple), generates weekly summary.
         Called by Celery Beat schedule.
