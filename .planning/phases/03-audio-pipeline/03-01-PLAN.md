  <action>
    1. Create backend/apps/audio/services/transcription.py:
       - Function `transcribe_narration(audio_file_path: str) -> dict`:
         Uses OpenAI `gpt-4o-transcribe` model, language='ko'
         Returns {'text': str, 'duration': float}
       - Function `transcribe_with_diarization(audio_file_path: str) -> dict`:
         Uses OpenAI `gpt-4o-transcribe-diarize` model, response_format='diarized_json', chunking_strategy='auto', language='ko'
         Returns {'text': str, 'duration': float, 'segments': list[dict]} where each segment has {speaker, text, start, end}
       - Use `from openai import OpenAI` with `settings.OPENAI_API_KEY` (NOT OpenRouter)
       - Handle errors: wrap in try/except, raise custom TranscriptionError

    2. Create backend/apps/audio/tasks.py:
       - @shared_task `transcribe_audio(recording_id: str)`:
         a. Load AudioRecording by id
         b. Based on recording_type: call transcribe_narration or transcribe_with_diarization
         c. For narration: create single TranscriptSegment with speaker='user'
         d. For live: create TranscriptSegment per segment from diarization result
         e. Update recording: status='completed', duration=result duration, full_text=joined text
         f. CRITICAL: In try/finally, always call recording.delete_audio_file() to remove temp audio
         g. On failure: set status='failed', error_message=str(error)

    3. Create backend/apps/audio/serializers.py:
       - AudioRecordingSerializer (id, recording_type, status, duration, full_text, emotion_intensity, post_action, created_at)
       - AudioRecordingCreateSerializer (validates audio file size <= 25MB, recording_type required)
       - TranscriptSegmentSerializer (id, speaker, speaker_label, text, start_time, end_time, emotion_intensity, order)
       - TranscriptSegmentUpdateSerializer (for editing speaker_label and text)
       - TranscriptDetailSerializer (recording info + nested segments)

    4. Create backend/apps/audio/views.py:
       - `upload_audio` (@api_view POST): Accept multipart form with 'audio' file and 'type' field.
         For 'live' type, also require 'consent_session_id' and validate RecordingConsent.Status.BOTH_CONSENTED.
         Save AudioRecording, save audio file, queue transcribe_audio.delay(). Return {recording_id, status: 'processing'}.
       - `recording_status` (@api_view GET): Return recording status, and if completed, include transcript segments.
       - `transcript_detail` (@api_view GET): Return full transcript with segments.
       - `update_segment` (@api_view PATCH): Allow editing segment text or speaker_label. Create TranscriptEdit audit record.
       - `assign_speakers` (@api_view POST): Bulk update speaker_label for all segments matching a speaker code.
         Body: {speaker_map: {"A": "나", "B": "파트너"}}. Updates all segments' speaker_label for the recording.
       - `delete_recording` (@api_view DELETE): Delete recording and all segments. Either partner can delete.
       - `set_post_action` (@api_view POST): Set post_action ('reframe', 'comfort', 'keep') for a recording.
         If 'reframe': call run_reframing_pipeline with full_text, save result to conversation.
         If 'comfort': call comfort endpoint logic (see below), save result.
         If 'keep': just save the action, no AI call.

    5. Create backend/apps/audio/urls.py:
       ```python
       urlpatterns = [
           path('upload/', upload_audio, name='audio-upload'),
           path('<uuid:recording_id>/status/', recording_status, name='audio-status'),
           path('<uuid:recording_id>/transcript/', transcript_detail, name='audio-transcript'),
           path('<uuid:recording_id>/segments/<uuid:segment_id>/', update_segment, name='audio-segment-update'),
           path('<uuid:recording_id>/speakers/', assign_speakers, name='audio-assign-speakers'),
           path('<uuid:recording_id>/action/', set_post_action, name='audio-post-action'),
           path('<uuid:recording_id>/', delete_recording, name='audio-delete'),
       ]
       ```

    6. Update backend/config/urls.py: Add `path('api/v1/audio/', include('apps.audio.urls'))`
