    1. Create mobile/src/features/recording/hooks/useLiveRecording.ts:
       - Manages the full live recording lifecycle: consent -> record -> upload
       - State: phase ('requesting_consent' | 'waiting_consent' | 'consent_granted' | 'consent_declined' | 'recording' | 'stopped')
       - Uses WebSocket to send consent request to partner (reuse existing consent WebSocket pattern from Phase 1)
       - requestConsent(): Opens WebSocket to consent channel, sends consent_request message with session_id
       - Listens for consent_response: if accepted, sets phase to 'consent_granted'; if declined, sets to 'consent_declined'
       - Consent timeout: 5 minutes (matches backend RecordingConsent model)
       - startRecording(): only callable when phase='consent_granted'. Uses useAudioRecording hook internally
       - stopRecording(): stops recording, gets URI. Either user can trigger this.
       - CRITICAL: Listen for 'stop_recording' WebSocket message from partner.
         When received, call stopRecording() locally to stop the actual audio capture.
         This is how partner's stop button affects the initiator's device.
       - sendStopToPartner(): sends 'stop_recording' message via WebSocket to partner's channel.
         Called automatically when initiator stops, so partner's UI also updates.
       - Stores consent_session_id for upload validation
       - Returns: { phase, requestConsent, startRecording, stopRecording, sendStopToPartner, consentSessionId, uri, metering, duration }

    2. Create mobile/src/features/recording/components/LiveConsentFlow.tsx:
       - Uses useLiveRecording hook
       - UI states:
         a. Requesting consent: "파트너에게 녹음 동의를 요청하고 있어요..." with loading spinner
            Show partner name if available, timer counting down from 5 minutes
         b. Waiting: Same as requesting, with pulsing animation
         c. Consent granted: "파트너가 동의했습니다!" brief success message, auto-proceed to recording
         d. Consent declined: "파트너가 녹음을 거부했습니다"
            Offer alternative: "혼자 기록하기로 전환할까요?" button -> switches to narration mode
            "취소" button -> goes back to mode selection
         e. Recording: Shows full RecordingControls + WaveformVisualizer (reuse from Plan 02)
            Persistent red bar at top: "녹음 중 - 파트너와 함께" with red dot
            Partner notification: "파트너도 녹음을 중지할 수 있습니다"
            Stop button calls both stopRecording() AND sendStopToPartner()
         f. Stopped: transition to RecordingPreview (reuse from Plan 02)
       - On submit: uploadAudio with type='live' and consentSessionId
       - After upload: poll status, navigate to transcript/[id] (live recordings go directly to transcript per mode-aware routing)

       - Listens for 'recording_stopped' WebSocket message to dismiss indicator
