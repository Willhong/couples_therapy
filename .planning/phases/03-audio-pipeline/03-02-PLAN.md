    2. Create mobile/src/features/recording/types.ts:
       ```typescript
       export type RecordingMode = 'narration' | 'live';
       export type RecordingStatus = 'idle' | 'recording' | 'paused' | 'stopped' | 'uploading' | 'processing' | 'completed' | 'failed';
       export type PostTranscriptAction = 'reframe' | 'comfort' | 'keep';

       export interface RecordingState {
         status: RecordingStatus;
         duration: number; // seconds
         uri: string | null;
         metering: number[]; // normalized 0-1 values
       }

       export interface TranscriptSegment {
         id: string;
         speaker: string;
         speaker_label: string | null;
         text: string;
         start_time: number;
         end_time: number;
         emotion_intensity: number | null;
         order: number;
       }

       export interface TranscriptResult {
         recording_id: string;
         status: string;
         duration: number | null;
         full_text: string | null;
         segments: TranscriptSegment[];
       }

       export interface GuidedPrompt {
         id: string;
         text: string;
         category: 'situation' | 'emotion' | 'need';
       }
       ```

    3. Create mobile/src/features/recording/hooks/useAudioRecording.ts:
       - Use expo-av Audio.Recording API
       - Manages recording lifecycle: start, stop, pause (if supported), cancel
       - CRITICAL: Call Audio.requestPermissionsAsync() before recording
       - CRITICAL: Call Audio.setAudioModeAsync({ allowsRecordingIOS: true, playsInSilentModeIOS: true }) before recording
       - Use Audio.RecordingOptionsPresets.HIGH_QUALITY with isMeteringEnabled: true
       - Poll metering data every 100ms via recording.getStatusAsync() during recording
       - Normalize metering from dB (-60 to 0) to 0-1 range: Math.max(0, (metering + 60) / 60)
       - Track duration via status.durationMillis / 1000
       - Enforce MAX_DURATION of 30 minutes (1800 seconds) - auto-stop
       - On stop: get URI from recording.getURI()
       - Expose: { startRecording, stopRecording, cancelRecording, recording state, metering data, uri, duration, isRecording }

    4. Create mobile/src/features/recording/hooks/useWaveform.ts:
       - Takes metering array (0-1 values), maxBars count (default 50)
       - Returns last `maxBars` values for rendering
       - Handles window sliding as new data comes in
       - Expose: { waveformData, addMeteringValue, reset }

    5. Create mobile/src/features/recording/services/audioApi.ts:
       - Import api from '@/lib/api'
       - `uploadAudio(uri: string, type: RecordingMode, consentSessionId?: string)`:
         Creates FormData with audio file (uri, type: 'audio/m4a', name: 'recording.m4a'),
         appends 'type' field, optionally 'consent_session_id'.
         POSTs to '/api/v1/audio/upload/' with Content-Type: multipart/form-data and 60s timeout.
         Returns { recording_id, status }
       - `pollTranscriptionStatus(recordingId: string, maxAttempts = 60, intervalMs = 2000)`:
         Polls GET '/api/v1/audio/{recordingId}/status/' every intervalMs.
         Returns TranscriptResult when status is 'completed'.
         Throws on 'failed' or timeout.
       - `getTranscript(recordingId: string)`: GET transcript detail
       - `updateSegment(recordingId: string, segmentId: string, data: {text?: string, speaker_label?: string})`: PATCH segment
       - `assignSpeakers(recordingId: string, speakerMap: Record<string, string>)`: POST speaker assignment
       - `setPostAction(recordingId: string, action: PostTranscriptAction)`: POST post-action
       - `deleteRecording(recordingId: string)`: DELETE recording

    6. Create mobile/src/features/recording/index.ts: barrel export for RecordingScreen and types
