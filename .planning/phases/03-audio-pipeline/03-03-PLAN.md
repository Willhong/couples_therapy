       - Re-export TranscriptSegment from recording/types
       - Add SpeakerMap type: Record<string, string> (e.g. {"A": "나", "B": "파트너"})
       - Add TranscriptState: { segments, speakerMap, isEditing, editingSegmentId }

    2. Create mobile/src/features/transcript/hooks/useTranscript.ts:
       - Takes recordingId: string
       - Loads transcript via getTranscript(recordingId) on mount
       - State: segments, speakerMap, loading, error, recordingType ('narration' | 'live')
       - Methods:
         - editSegment(segmentId, data): calls updateSegment API, updates local state
         - updateSpeakerMap(map): calls assignSpeakers API, updates all speaker_labels locally
         - chooseAction(action): calls setPostAction API
       - Returns: { segments, speakerMap, loading, error, recordingType, editSegment, updateSpeakerMap, chooseAction }

    3. Create mobile/src/features/transcript/hooks/useAudioPlayer.ts:
       - Takes audioUri: string | null (may be null if audio already deleted from server)
       - Uses expo-av Audio.Sound API
       - State: isPlaying, position (ms), duration (ms)
       - Methods:
         - play(), pause(), seekTo(positionMs)
         - jumpToTime(seconds): converts to ms, seeks
       - Handles Sound lifecycle: load on mount, unload on unmount
       - Updates position via onPlaybackStatusUpdate callback
       - If audioUri is null: all methods are no-ops, UI shows "오디오가 삭제되었습니다"
       - Returns: { isPlaying, position, duration, play, pause, seekTo, jumpToTime, isAvailable }

    4. Create mobile/src/features/transcript/components/TranscriptLine.tsx:
       - Props: segment (TranscriptSegment), speakerLabel (string), isCurrentUser (boolean), isPlaying (boolean for highlight), onPress, onLongPress
       - Chat bubble style:
         - If isCurrentUser (speaker matches current user): right-aligned, blue-ish background (#EBF5FF)
         - Else: left-aligned, gray background (#F3F4F6)
       - Shows speaker_label (or speaker code if no label) at top of bubble
       - Shows text content in bubble body
       - Shows timestamp (start_time formatted as MM:SS) below bubble
       - Emotion intensity: subtle left border color (1=light blue -> 5=deep red gradient)
       - onPress: jump to audio position at segment.start_time
       - onLongPress: open edit modal (pass up via callback)
       - isPlaying prop: when true, add subtle highlight border to show currently playing segment

    5. Create mobile/src/features/transcript/components/AudioPlayer.tsx:
       - Props: useAudioPlayer hook return values
       - Horizontal bar with play/pause button, progress bar, time display
       - Progress bar: Pressable, tapping a position seeks to that time
       - Time: current / total in MM:SS format
       - Compact design, sits at top or bottom of transcript view
       - If not available: show disabled state with "오디오 없음" text

    6. Create barrel export in index.ts
