<objective>
Replace the 5-call sequential LangGraph pipeline with a single-call architecture for dramatically faster reframing responses.

Purpose: The current pipeline runs safety_check -> acknowledge -> analyze -> suggest -> combine (5 sequential LLM calls), making each message take 15-30 seconds. The new architecture collapses this into 1 LLM call with a well-crafted system prompt that outputs structured JSON, plus a regex-based safety pre-filter (zero LLM calls for safety check). This should reduce response time by ~80%.

Output: Rewritten reframing_graph.py (single-call pipeline), consolidated system_prompts.py, and updated views.py. The response JSON format stays identical so frontend code requires zero changes.
</objective>

**Rewrite reframing_graph.py** to replace the entire LangGraph StateGraph with a simple single-call pipeline. Remove all LangGraph imports and node functions.

The new module should contain:

1. **`check_safety(message: str) -> dict | None`** (sync, no LLM call):
   - Uses regex matching against SAFETY_KEYWORDS from system_prompts
   - For "severe" keyword matches: return SAFETY_RESPONSE_TEMPLATE (immediate, no LLM)
   - For "mild" keyword matches: return `{"mild_flag": True}` (LLM will still process, but flag is noted)
   - For no matches: return None
   - Use `re.search` with word boundaries where possible, but Korean doesn't have word boundaries so use simple `in` check on the message text

2. **`async def run_reframing_pipeline(user_message: str, conversation_context: str = "") -> dict`**:
   - Step 1: Run `check_safety(user_message)`. If severe abuse detected, return immediately with safety template. Set `is_abuse_detected = True`.
   - Step 2: Build messages list for single LLM call:
     - SystemMessage with REFRAMING_SYSTEM_PROMPT
     - HumanMessage with formatted input: conversation context (if any) + user message
   - Step 3: Call `get_chat_model(temperature=0.6, streaming=False)` and `await model.ainvoke(messages)`
   - Step 4: Parse JSON response using `_parse_json_response()` (keep existing JSON parser, it handles markdown code blocks well)
   - Step 5: Build return dict matching the EXACT format views.py expects:
     ```python
     {
         'final_response': parsed.get('acknowledgment', '') + '\n\n' + _build_readable_response(parsed),
         'analysis': parsed.get('analysis'),
         'suggestions': parsed.get('suggestions', []),
         'is_abuse_detected': False,
         'safety_response': None,
     }
     ```
   - The `_build_readable_response(parsed)` helper should create a natural Korean text from the structured analysis (for the chat bubble content), similar to what the old combine node did but without an LLM call. Format:
     ```
     [공감 메시지]

     당신이 말한 것을 상대방은 이렇게 들었을 수 있어요:
     [how_they_heard]

     왜 이런 차이가 생겼을까요:
     [why_the_gap]

     다음에 시도해볼 것:
     - [suggestion 1]
     - [suggestion 2]
     - [suggestion 3]
     ```
   - If mild safety flag was set, add it to the response but still process normally

3. **`def run_reframing_pipeline_sync(user_message: str, conversation_context: str = "") -> dict`**:
   - Sync wrapper using `asyncio.run()` for the async version
   - Keep this for backward compatibility

4. **Remove entirely:**
   - All LangGraph imports (StateGraph, END)

   - `_parse_json_response()` helper (it works well for extracting JSON from LLM responses)
